# Sprint 1 - Project Foundation

This folder explains the work completed in the first sprint of MediLink in non-technical language. Each page describes what was built, why it matters, and what the project can do now.

## Documents

- `ham-5-workspace-structure.md` - how the repository was organized and prepared for future work
- `ham-6-backend-foundation.md` - how the backend was started and how it responds today
- `ham-7-frontend-foundation.md` - how the Angular app was prepared for role-based screens and API calls
- `ham-8-initial-mysql-schema.md` - how the initial MVP database schema was designed
- `ham-9-database-migrations-and-seed-data.md` - how Flyway migrations and local seed data were configured
- `../database-schema.md` - database schema decisions from HAM-8
- `../database-migrations.md` - migration and seed data behavior from HAM-9

## How To Read These Docs

Each document follows the same pattern:

- what the issue was trying to achieve
- what was actually added
- what this means for non-developers
- what is still intentionally unfinished

That keeps the docs useful for product, design, and project tracking without requiring code knowledge.
